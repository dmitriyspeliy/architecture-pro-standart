# Task 1. Схема интеграции приложений

## Цель
Показать текущие взаимодействия между участниками процессов и IT-системами банка для депозитного и кредитного процессов.

## Процессы
- Открытие депозита / накопительного счёта
- Открытие кредита

## Ключевые участники
- Клиент
- Сотрудник отделения
- Оператор кол-центра
- Сотрудник депозитного бэк-офиса
- Сотрудник кредитного бэк-офиса

## Основные интеграции
- Кол-центр → АБС
- АБС → СМС-шлюз
- АБС ↔ Интернет-банк (интеграция через БД)
- АБС → Кредитный конвейер (пакетно, раз в сутки)
- Кредитный конвейер → Система скоринга
- Система скоринга → БКИ API

## Проблемные места
- Email как часть основного бизнес-процесса
- Excel как инструмент расчёта ставок
- Пакетные интеграции вместо событийных или API
- Высокая зависимость от ручной работы сотрудников
